## ✨ ฟีเจอร์หลัก (Key Features)

### 1. 📖 ระบบทบทวนบทเรียนแบบโต้ตอบ (Interactive Review)
* **Simulator 1: แหล่งกำเนิดความดันแก๊ส** - แบบจำลอง Canvas 2D ที่แสดงให้เห็นการเคลื่อนที่ชนผนังของโมเลกุลแก๊สซึ่งก่อให้เกิดความดัน ($P$) ผู้เรียนสามารถปรับจำนวนโมเลกุล ($N$) และอุณหภูมิ/อัตราเร็ว เพื่อดูผลกระทบต่ออัตราการชนและความดันได้แบบ Real-time
* **Simulator 2: พลังงานจลน์เฉลี่ย** - อธิบายความสัมพันธ์ที่อุณหภูมิแปรผันตรงกับพลังงานจลน์เฉลี่ย ($\bar{E}_k$) โดยไม่ขึ้นกับชนิดของแก๊ส
* **Simulator 3: เปรียบเทียบอัตราเร็ว RMS** - แอนิเมชันเปรียบเทียบความเร็วของแก๊สต่างชนิดที่อุณหภูมิเดียวกัน (แก๊สเบา เช่น ไฮโดรเจน จะเคลื่อนที่เร็วกว่าแก๊สหนัก เช่น แก๊สที่เลือก)
* **เครื่องคิดเลขคำนวณ $v_{rms}$** - เครื่องมือคำนวณที่แสดงขั้นตอนการแทนค่าตัวเลขลงในสมการ (Substitution) ด้วยฟอร์แมตคณิตศาสตร์ที่สวยงามและรองรับการเปลี่ยนค่าแบบเรียลไทม์

### 2. 🧠 ระบบตะลุยโจทย์ (Dynamic Practice Mode)
* **ระบบสุ่มตัวแปรในโจทย์ (RNG)** - มีโหมดให้เลือกทั้งโจทย์มาตรฐาน (ตัวเลขตามหนังสือเรียน) และโจทย์สุ่มตัวแปร (Non-deterministic) เพื่อให้ผู้เรียนได้ฝึกการคำนวณตัวเลขที่ไม่ซ้ำกัน
* **คลังโจทย์และการค้นหา/กรอง** - สามารถเลือกฝึกฝนโจทย์แยกตามหัวข้อย่อย 16.3.1 (ความดัน), 16.3.2 (พลังงานจลน์เฉลี่ย), 16.3.3 (อัตราเร็ว RMS) และแสดงจำนวนข้อสอบทั้งหมดในคลัง
* **ระบบตรวจคำตอบทันที** - พร้อมเฉลยวิธีทำทีละขั้นตอน (Step-by-step Explanation) ที่ใช้สมการ LaTeX ที่อ่านง่าย

### 3. ⏱️ ระบบสอบเก็บคะแนน (Timed Exam System)
* **ระบบจำลองการสอบแบบจำกัดเวลา** - จำนวน 5 ข้อ (คะแนนเต็ม 10 คะแนน) โดยมีระยะเวลาจำกัดในการทำข้อสอบ **15 นาที** พร้อมตัวจับเวลาถอยหลัง (Timer) ในแถบเมนูด้านบน
* **การสอบแบบผสมผสาน** - ระบบจะสุ่มข้อสอบโดยมี **ข้อสอบแบบเลือกตอบ (Choice) เพียง 1 ข้อ** และ **ข้อสอบแบบคำนวณตัวเลข (Numeric Input) อีก 4 ข้อ** เพื่อวัดผลทั้งความเข้าใจเชิงทฤษฎีและทักษะการคำนวณอย่างรอบด้าน
* **ระบบป้องกันการโกง (Anti-cheat guard)** - แจ้งเตือนและป้องกันการรีเฟรชหน้าจอหรือออกจากหน้าสอบโดยไม่ตั้งใจ
* **Dashboard สรุปผล** - แสดงคะแนนรวม เวลาที่ใช้ พร้อมตารางสรุปสถานะการตอบแต่ละข้อ และปุ่มดูเฉลยละเอียดวิธีคิดอย่างเป็นขั้นเป็นตอน
* **ระบบบันทึกผลคะแนน** - บันทึกผลสอบล่าสุดเก็บไว้ใน Local Storage ของเบราว์เซอร์ เพื่อให้ผู้เรียนสามารถตรวจสอบคะแนนล่าสุดได้ที่หน้าแรก

## 🎯 กฎระบบข้อสอบแปรผัน (Dynamic Quiz System Rules)

ระบบข้อสอบในโปรเจกต์นี้ได้รับการออกแบบให้เป็น **Non-Deterministic Dynamic System** โดยผูกค่าตัวแปรในโจทย์เข้ากับ **"เลขที่ของนักเรียน (Roll Number: $R$)"** เพื่อให้มั่นใจว่านักเรียนแต่ละคนจะได้รับพารามิเตอร์โจทย์และคำตอบสุดท้ายที่ไม่ซ้ำกัน

### 📌 สถาปัตยกรรมและลอจิกการทำงาน:
1. **Dynamic Parameter Generation:** ระบบจะดึงเลขที่ของนักเรียน ($R$ ตั้งแต่ 1-40) มาคำนวณร่วมกับฐานตัวแปรที่ถูกสุ่มขึ้นมาแบบอิสระ (Non-deterministic Random) เช่น `T = T_base + R` หรือบวกเลขที่เข้าไปในพารามิเตอร์ เพื่อสร้างค่าพารามิเตอร์ชุดใหม่สำหรับโจทย์แต่ละข้อ
2. **On-the-fly Validation:** การประเมินคำตอบและเฉลยวิธีทำจะคำนวณแบบสดๆ (On-the-fly) จากเลขที่ $R$ ของผู้เข้าสอบจริง โดยไม่ใช้ค่าคงที่ตายตัว (No Static Hardcoded)
3. **Safety Constraints:** สมการเบื้องหลังได้รับการกำหนดขอบเขตค่าต่ำสุด-สูงสุดไว้อย่างรัดกุม เพื่อป้องกันไม่ให้การคำนวณร่วมกับเลขที่ $R$ เกิดค่าที่ติดลบหรือไม่สมเหตุสมผลทางฟิสิกส์

## 🛠️ เทคโนโลยีที่ใช้ (Tech Stack)

* **HTML5**: สำหรับจัดวางโครงสร้างและเนื้อหาเว็บแอปพลิเคชัน
* **Vanilla CSS (style.css)**: สำหรับการตกแต่งเพิ่มเติม เช่น จัดการแอนิเมชันการวิ่งของอนุภาค ปรับแต่ง Scrollbar และเอฟเฟกต์การคลิกปุ่ม
* **JavaScript (ES6+ / app.js)**: สำหรับควบคุมลอจิกทั้งหมด ระบบคำนวณฟิสิกส์ อัลกอริทึมการสุ่มและตรวจข้อสอบ ตลอดจนตัวจับเวลา
* **Tailwind CSS (via CDN)**: ใช้ในการจัดการ UI/UX, สีสัน, รูปแบบ Grid/Flex Layout และความ Responsive รองรับทั้งคอมพิวเตอร์ แท็บเล็ต และสมาร์ทโฟน
* **MathJax (via CDN)**: ไลบรารีสำหรับเรนเดอร์สมการคณิตศาสตร์และสูตรฟิสิกส์ด้วยฟอร์แมต LaTeX
* **FontAwesome**: บริการไอคอนที่ใช้แสดงสัญลักษณ์ในหน้าเว็บ
* **Google Fonts**: นำเข้าฟอนต์ 'Prompt' และ 'Sarabun' เพื่อให้อ่านง่าย สบายตา และดูมีความเป็นสากลและวิชาการ

## 🚀 วิธีการใช้งาน (How to Run)

เนื่องจากโปรเจกต์นี้ทำงานแบบ Client-side ทั้งหมด คุณจึงไม่จำเป็นต้องติดตั้งโปรแกรมหรือเซิร์ฟเวอร์แบ็กเอนด์ใดๆ
1. ตรวจสอบให้มั่นใจว่าไฟล์ `index.html`, `app.js`, และ `style.css` อยู่ในโฟลเดอร์เดียวกัน
2. เปิดไฟล์ `index.html` ด้วยเว็บเบราว์เซอร์ทั่วไป (เช่น Google Chrome, Microsoft Edge, Safari, Firefox)
3. หากเปิดใช้งานผ่านเครื่องมือเช่น Live Server บน VS Code หรือโฮสต์บนเว็บเซิร์ฟเวอร์แบบ Static จะช่วยลดปัญหาเรื่อง File Paths
4. พร้อมใช้งานทันที! *(จำเป็นต้องเชื่อมต่ออินเทอร์เน็ตในการโหลดฟอนต์และไลบรารี CDN ในครั้งแรก)*
